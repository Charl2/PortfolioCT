Project Overview

Automating CV Data Extraction

This project improved an existing CV scraping automation in Zapier, designed to read CV names from a Drive folder, match them with the company's HubSpot database, and extract relevant information using GPT prompts for display in HubSpot contact forms.

The workflow required clear formatting of CV file names, such as "CV WEEM - First NAME - Fr/En," for proper identification in HubSpot. Integrating the GPT-4o API improved data extraction, with automated scripts minimising errors. An email alert system was in place to notify the team of any processing issues.

Key Challenges

Some challenges included managing access between Drive and Zapier, ensuring GPT-4o returned only the required output, and dealing with varied CV formats. HubSpot connections would fail when hyphens were used incorrectly (e.g., "CV-WEEM - Name"), when names were reversed (e.g., "Surname Firstname"), or when accents were missing (e.g., "Marie Clenet" instead of "Marie Clénet").

Conclusion

This project showed that reliable automation requires as much attention to edge cases as to the happy path. By iteratively refining GPT prompts, adjusting scripts for varied CV layouts, and building a robust error alerting system, the pipeline moved from a frequent source of manual debugging to a dependable intake process — saving the team significant time each week.
