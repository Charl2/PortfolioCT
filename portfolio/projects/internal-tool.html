My Role

UX Designer, WEEM

I redesigned an internal tool for a company's HR department to search and select top freelancers for various missions. The original AI-powered tool, coded in Python and displayed through Streamlit, had visual and usability limitations. My goal was to create a more intuitive and engaging interface tailored for HR professionals.

Project Overview

The main search page allowed seamless filtering by skills, positions, sectors, expertise, availability, and budget through an easy-to-use sidebar. The results section displayed essential details like job titles, skills, and availability percentages, enabling quick comparisons.

The detailed consultant focus page offered in-depth profiles with background, key projects, and expertise, helping HR teams assess suitability at a glance. The redesign emphasised user-centric navigation, clear typography, and intuitive layouts — transforming the tool into an effective, user-friendly platform that met HR needs.

Experience Overview

Three Views, One Tool

📋
List View Presents essential details such as names, roles, ratings, and mission match percentages, enhanced with colour-coded labels and icons highlighting key attributes like top consultant status.
🗂️
Grid View Larger profile cards with photos, ratings, roles, and key experience highlights for a visual overview.
👤
Freelancer Focus Page Detailed profiles with background, key projects, and expertise for informed decision-making after selection.

Conclusion

Redesigning this internal AI tool required balancing information density with usability — making sure HR professionals could quickly scan, compare, and drill into consultant profiles without cognitive overload. By introducing clear visual hierarchy, colour-coded status tags, and dedicated view modes, the redesign transformed a functional but limited Streamlit interface into an effective, user-friendly platform.
